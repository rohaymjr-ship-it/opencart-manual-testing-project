OpenCart E-Commerce Website Test Plan
1. Introduction

This document describes the testing strategy, scope, objectives, resources, and schedule for testing the OpenCart E-Commerce Demo Website.

Application Under Test (AUT):

OpenCart Demo

The purpose of this testing activity is to ensure that the core functionalities of the website are working correctly and meet the expected business requirements.

2. Objectives

The main objectives of testing are:

Verify that all major website functionalities work correctly.
Identify defects and usability issues.
Validate user workflows such as product browsing, cart management, and checkout.
Ensure proper UI behavior and responsive design.
Improve application quality and reliability.
3. Scope of Testing
In Scope

The following modules will be tested:

User Interface
Home page
Navigation menu
Footer and header links
User Account
User registration
User login
Logout functionality
Password recovery
Product Module
Product search
Product details
Product categories
Product filtering
Shopping Cart
Add to cart
Remove from cart
Update product quantity
Checkout Module
Guest checkout
Shipping details
Payment methods
Order confirmation
Responsive Testing
Desktop view
Tablet view
Mobile view
Out of Scope

The following areas are excluded from testing:

Performance testing
Security testing
Database testing
Third-party payment gateway validation
Backend admin panel testing

The following testing types will be performed:

Functional Testing

UI Testing

Responsive Testing

Testing will begin when:

The application is accessible.
Test environment is ready.
Required test data is available.
8. Exit Criteria

Testing will be completed when:

All critical and high-severity defects are resolved.
Major functionalities pass testing.
Test execution is completed.
Test summary report is prepared.
